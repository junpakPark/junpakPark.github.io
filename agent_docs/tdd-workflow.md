# TDD 워크플로우

## 코딩 페르소나
- **TDD 중심**: 테스트 먼저, 작은 단계로 진행
- **미니멀리즘**: 필요한 것만 구현, 과도한 설계 금지 (YAGNI)
- **점진적 개선**: Simple Design → Refactoring 사이클

## TDD 사이클 (Red-Green-Refactor)

각 기능 단위로 아래 사이클을 반복하고, 사이클 완료 시 커밋:

```
1. Red     → 실패하는 테스트 작성
2. Green   → 테스트를 통과하는 최소한의 구현
3. Refactor → 코드 정리 (테스트는 계속 통과)
4. Commit  → 해당 사이클의 변경사항 커밋
```

## 설계 제약
- 작은 특화 컴포넌트 (하나의 역할)
- 합성(Composition) 우선
- 함수형 스타일 (순수 함수, 불변성)
- React 우선, 현재 패턴 유지

## 금지 패턴
- 과도한 추상화 (미래를 위한 불필요한 설계)
- 전역 상태 남용
- `any` 타입 사용

## 품질 우선순위
1. 타입 안전성 (TypeScript strict)
2. 접근성 (시맨틱 HTML, aria-labels)
3. SEO (메타 태그)
4. 성능 (번들 최적화)
